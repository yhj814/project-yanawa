# project-yanawa

<h1> 대한민국 청년들의 스포츠 대회 관리 플랫폼 </h1>

<h2>1. 기획 배경</h2>

현대 청년들이 스포츠 활동을 통해 스트레스를 해소하고 건강을 증진하며, 삶의 행복을 느낄 수 있는 기회를 제공하여 청년들이 더욱 활발하게 스포츠 활동에 참여하고 사회적 관계망을 형성할 수 있도록 돕고자 기획하게 되었다.

<h2>2. 기대 효과</h2>

* 스포츠 참여 활성화

스포츠를 즐기고자 하는 청년들이 팀원 모집과 팀 매칭 과정을 간소화하여 더욱 쉽게 참여할 수 있도록 도와줍니다.
팀 관리와 경기 기록 관리 기능을 통해 활동 데이터를 직관적으로 확인할 수 있습니다. <br>

* 커뮤니티 활성화

개인 간 혹은 팀 간의 자유게시판과 문의하기 기능을 통해 사용자가 상호 소통할 수 있는 장을 제공합니다.
스포츠 활동을 기반으로 하는 새로운 인맥 형성 및 협력 기회를 증대시킵니다. <br>

* 운영 효율성 증대

장소 대여와 경기 매칭 프로세스를 효율적으로 관리하여 사용자는 필요한 정보를 빠르게 확인하고 즉시 예약 및 신청할 수 있습니다.
관리자 페이지를 통해 사용자 활동을 종합적으로 관리하여 사이트 운영의 효율성을 높입니다. <br>

* 사회적 가치 창출

청년들이 스포츠를 통해 건강한 라이프스타일을 유지하고, 새로운 사람들과 협력하며 사회적 관계망을 확장할 수 있는 기회를 제공합니다.
전국적으로 스포츠 활동의 접근성을 개선하여 스포츠 문화 확산에 기여합니다. <br><br>

1. 세션(HttpSession)의 이해:

세션은 서버 측에서 유지되는 사용자별 저장 공간으로, 각 사용자에 대한 정보를 저장할 수 있고 세션에 데이터를 저장하면 해당 사용자가 웹 애플리케이션을 이용하는 동안 그 데이터를 계속 사용할 수 있다.
세션에 객체 저장:setAttribute 메서드를 사용하여 객체를 세션에 저장할 수 있고 이때 키 값(여기서는 member)를 통해 나중에 세션에서 해당 객체를 가져올 수 있었다.
예외 처리와 흐름 제어:

로그인 실패 시 예외를 발생시켜 흐름을 제어하는 부분이 복잡했고 예외가 발생하면 그 이후의 코드가 실행되지 않으므로, 로그인 실패 시 예외를 잡아 처리했다.

<img src="https://github.com/user-attachments/assets/1719dfe2-e248-495f-91f3-c1d3d5f59750"> <br>
🚀로그인 실패시 LoginFailException 으로 처리! 🚀 <br><br>


2. 쿠키의 개념과 사용법:

쿠키는 클라이언트 측에 저장되는 작은 데이터로, 서버와 클라이언트 간의 상태를 유지하는 데 사용된다. 쿠키를 생성하고 설정하며, 응답에 추가하여 클라이언트에 전달하는 과정이 복잡했다.
쿠키 유효 기간 설정: setMaxAge 메서드의 매개변수에 따라 쿠키의 유효 기간이 달라지고 -1: 세션 쿠키로, 브라우저가 종료되면 삭제, 0: 즉시 삭제된디. 로그인 유지를 위해서는 브라우저 종료 후에도 쿠키가 유지되어야 하므로, 적절한 양의 정수를 설정해야 한다.

<h3>6-2 문제점과 추가 보완 사항</h3> <br>

🚨 <strong>문제점</strong> 🚨<br>
비밀번호를 쿠키에 평문으로 저장하는 문제
비밀번호를 쿠키에 평문으로 저장하면, 클라이언트 측에서 쿠키를 탈취당할 경우 비밀번호가 그대로 노출될 위험이 있다.<br>
🚀 보완 사항🚀<br>
비밀번호를 쿠키에 저장하지 않고 로그인 유지 기능을 위해서는 토큰 기반 인증 방식을 사용, 서버에서 고유한 토큰을 생성하여 쿠키에 저장하고, 이 토큰을 통해 사용자를 인증하도록 보완한다.<br><br>

🚨 <strong>문제점 </strong>🚨<br>
쿠키 삭제 시 값 설정 문제
쿠키를 삭제할 때 값을 null로 설정했다. 일부 브라우저에서는 쿠키 값이 null일 때 예외가 발생할 수 있다.<br>
🚀 보완 사항🚀<br>
쿠키의 값을 그대로 두고 setMaxAge(0)만 설정하여 쿠키를 삭제한다. <br><br>

🚨 <strong> 문제점</strong>🚨<br>
로그인 유지 기능의 안전성 문제
현재 로그인 유지 기능은 비밀번호를 쿠키에 저장하는 등 안전하지 않다.<br>
🚀 보완 사항🚀<br>
토큰 기반 인증 방식을 도입하여 로그인 시 서버에서 고유한 토큰을 생성하고, 이 토큰을 쿠키에 저장하여 인증에 사용, 토큰은 적절한 만료 시간을 설정하고, 서버 측에서 관리하여 보안을 강화한다. <br><br> <hr>

<h4>🌟 첫 프로젝트  </h4>

사용자의 편의 보단 내가 개발하기 편한 상황을 만들었고 그 결과 내가 개발한 서비스를 내가 이용할때 마저 불편하고 이건 왜 이렇게 만들었을까? 라는 생각이 들었다. 다음 프로젝트에서는 어렵고 힘들었던 상황을 토대로 더 발전한 프로젝트를 진행해야겠다.

<h4>🌟 내것만 완성된다고 끝난것이 아니다.   </h4>
이번 프로젝트때 난 퍼블리싱과 백엔드 내가 맡은 부분을 모두 해냈지만, 못해낸 팀원들도 있어 완벽한 서비스를 구현하는 것에 문제점이 있었다. 중간에 그만둔 팀원도 있었기에 구현 안된 서비스의 빈자리는 더욱 크게 느껴졌고 일단 내거라도 마무리해보자 하는 생각으로
남은 기간동안 정말 열심히 했던거 같다. 다음 프로젝트에는 내 것을 얼른 끝낸 후 다른 팀원들을 도와 더 나은 프로젝트를 만들어보고 싶다.
